## 2. Agentic AI 개념 정리

2026년 5월 기준 생성형 AI(Generative AI)는 거대언어모델(Large Language Model, LLM)이 질문에 답하거나 문서·코드·이미지를 생성하는 능력으로 이해된다. 이에 비해 에이전틱 AI(Agentic AI)는 목표를 입력받은 뒤 필요한 절차를 나누고, 외부 도구를 호출하며, 중간 결과를 보고 다음 행동을 조정하는 실행 체계이다. 따라서 차이는 산출물의 형식보다 통제 방식에 있다. 생성형 AI가 응답 중심이라면, 에이전틱 AI는 제한된 범위 안에서 업무 흐름을 운영하는 시스템에 가깝다.[^1]

에이전틱 AI의 핵심 특성은 네 가지로 정리된다. 계획(Planning): 복수의 하위 과업을 순서화한다. 예컨대 계약서 검토를 조항 식별, 위험 표시, 수정안 작성으로 나눈다. 도구사용(Tool Use): 검색, 데이터베이스, 업무 소프트웨어, 코드 실행기를 호출한다. 예컨대 회계 자료를 조회한 뒤 스프레드시트에 검산 결과를 입력한다. 다단계실행(Multi-step Execution): 한 번의 답변이 아니라 관찰과 수정의 반복으로 과업을 끝낸다. 예컨대 판례 검색 결과가 부족하면 검색어를 바꾸어 다시 탐색한다. 자율성(Autonomy): 사람이 정한 목표와 제약 안에서 다음 행동을 선택한다. 예컨대 오류가 난 API 호출을 다른 경로로 재시도한다. 이 자율성은 완전한 독립이 아니라 비용, 오류, 책임 문제 때문에 통제 장치와 함께 설계된다.[^2]

주요 프레임워크도 이 차이를 반영한다. LangGraph는 상태 보존, 중단, 사람 승인, 장기 실행에 강하므로 감사 보조나 리서치처럼 추적성이 필요한 업무에 적합하다. 다만 저수준 제어가 많아 초기 설계 부담이 있다. AutoGen은 여러 에이전트의 대화와 협업 실험에 강하며 연구·프로토타입 제작에 유리하다. 그러나 운영 단계에서는 관찰성, 배포, 유지보수 설계가 별도 과제가 된다. CrewAI는 역할 기반 에이전트와 플로를 빠르게 구성하기 쉬워 마케팅 조사, 고객 대응, 반복 보고 자동화에 알맞다. 반면 복잡한 예외 처리나 엄격한 감사 추적에는 추가 설계가 필요하다.[^3]

이 보고서는 에이전틱 AI를 단일 제품명이 아니라 업무 수행 구조로 본다. 즉 Agentic AI는 단일 모델이 아니라 모델·도구·메모리·계획 모듈의 결합체이며, 이 결합 방식이 산업 도입의 변수가 된다. 회계·법무·컨설팅은 모두 지식서비스 산업이지만 규제, 책임, 정형성, 고객 구조, 자격 장벽이 다르다. 따라서 같은 기술도 어느 모듈을 열고 닫을지에 따라 산업별 도입 속도와 허용 범위가 달라진다.[^4]
